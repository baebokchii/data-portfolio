# 배송 지연 분석 프로젝트

## 1. 프로젝트 개요

본 프로젝트는 브라질 이커머스 플랫폼 Olist의 주문 데이터를 활용하여 배송 지연이 고객 만족도에 어떤 영향을 미치는지를 분석한 프로젝트입니다. 

주문 정보, 배송 일정, 고객 리뷰 데이터를 주문 단위로 결합하여 배송 성과와 고객 반응 간의 관계를 수치적으로 확인하였으며, 이를 통해 이커머스 운영 관점에서 활용 가능한 인사이트를 도출하였습니다. 

프로젝트 기간 : 2025년 12월 26일 ~ 현재

활용 데이터 셋 : [Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

활용 언어 : SQL, Python

시각화 툴 : Python, Tableau

## 2. 분석 배경과 문제 정의

Olist는 다수의 판매자가 참여하는 마켓플레이스 구조의 이커머스 플랫폼입니다. 
이러한 구조에서는 판매자와 지역에 따라 배송 성과가 달라질 가능성이 높으며, 배송 품질 관리가 플랫폼 전체의 고객 경험에 중요한 요소로 작용합니다. 

배송 지연이 반복적으로 발생할 경우 고객 만족도가 하락할 수 있고, 이는 리뷰 점수 저하와 함께 판매자 및 플랫폼에 대한 신뢰도 감소로 이어질 수 있다.

이에 따라 본 프로젝트에서는 다음과 같은 질문을 중심으로 분석을 진행하였습니다. 
- 배송 지연이 고객 리뷰 점수에 부정적인 영향을 미치는가
- 배송 지연이 특정 지역에서 더 빈번하게 발생하는가

## 3. 데이터 소개

본 프로젝트에서는 Kaggle에 공개된 Olist 이커머스 데이터셋을 사용하였습니다. 

해당 데이터셋은 2016년부터 2018년까지 약 10만 건의 주문 정보를 포함하고 있으며, 주문 단위로 배송 정보와 고객 리뷰를 연결할 수 있는 구조를 가지고 있습니다.

이러한 특성으로 인해 배송 지연과 고객 만족도 간의 관계를 직접적으로 분석하는 데 적합한 데이터셋입니다.

분석에 사용한 테이블은 다음과 같습니다.

- olist_customers_dataset
- olist_geolocation_dataset
- olist_order_items_dataset
- olist_order_payments_dataset
- olist_order_reviews_dataset
- olist_orders_dataset
- olist_products_dataset
- olist_sellers_dataset
- product_category_name_translation

각 테이블의 상세 스키마는 data 폴더에 정리하였습니다. 

## 4. 분석 질문 정의

배송 지연 여부는 다음과 같이 정의하였습니다. 

- 지연 주문 : 실제 배송 완료일이 예상 배송일보다 늦은 주문
- 정상 주문 : 실제 배송 완료일이 예상 배송일보다 빠르거나 같은 주문

위 정의를 바탕으로 다음 질문을 검증하였습니다. 

- 지연 주문과 정상 주문 간의 평균 리뷰 점수는 차이가 있는가
- 판매자 지역 및 구매자 지역에 따라 배송 지연 비율은 다른가

## 5. 데이터 전처리

분석의 정확도를 확보하기 위해 다음과 같은 전처리 과정을 수행하였습니다. 

- 주문 데이터와 리뷰 데이터는 order_id 기준으로 결합
- 예상 배송일과 실제 배송일 등 배송 관련 날짜 컬럼은 모두 yyyy-mm-dd 형식으로 통일
- 실제 배송 완료일과 예상 배송일이 모두 존재하는 주문만 분석 대상에 포함
- 리뷰 점수가 존재하지 않는 주문은 고객 만족도 분석에서 제외
- 배송 지연 여부를 나타내는 boolean 컬럼을 생성

배송 일정 정보가 불완전한 주문은 분석 결과 왜곡 가능성이 있어 제외하였습니다. 

## 6. 분석 방법

- 배송 지연 여부에 따른 리뷰 점수 분포 비교
- 평균 리뷰 점수 차이를 계산
- 지역별 배송 지연 비율을 계산 및 비교

**해당 프로젝트는 위의 project scope대로 진행할 예정이며, result 파일에 분석 결과 및 SQL 쿼리, 인사이트, 한계와 배운점 등을 서술할 예정입니다.**
